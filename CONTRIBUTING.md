# Правила внесения изменений

1. Одно логически завершённое изменение оформляется отдельным коммитом.
2. В описании указываются затронутые индикаторы, КИМ и веса.
3. Новый КИМ должен содержать назначение, условия, задание, формат сдачи, критерии, шкалу и правила использования внешних ресурсов и генеративного ИИ.
4. Максимум КИМ должен совпадать с итоговой таблицей и `data/competency_map/mapping.csv`.
5. Перед публикацией запускается `python tools/validate_repository.py`.
6. Для значимых изменений рекомендуется отдельная ветка `feature/<название>` или `fix/<название>`.

Рекомендуемые сообщения коммитов: `docs: ...`, `kim: ...`, `rubric: ...`, `fix: ...`.
