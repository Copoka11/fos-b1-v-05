# КИМ-2.1. ЛР5 — MLP для классификации табличных данных

| Параметр | Значение |
|---|---|
| Индикаторы | DL-1.1, DL-1.2 |
| Уровень | С, С |
| Максимум | 8 баллов |
| Формат | индивидуальная работа; код PyTorch, конфигурация и отчёт |

## Цель

Реализовать обучение MLP, обосновать архитектуру, функцию потерь и оптимизатор и честно сравнить нейросеть с классическим baseline.

## Обязательное задание

1. Выбрать табличный классификационный датасет и подготовить единые train/validation/test для всех моделей.
2. Реализовать MLP с 2–3 скрытыми слоями, задав активации, инициализацию и регуляризацию.
3. Для `CrossEntropyLoss` либо `BCEWithLogitsLoss` передавать сырые logits: не добавлять `Softmax`/`Sigmoid` перед функцией потерь.
4. Реализовать train/validation loop, фиксировать seed, сохранять лучшую эпоху по validation и применять early stopping.
5. Сравнить SGD с momentum и Adam либо обосновать другой выбор оптимизаторов.
6. Сопоставить MLP с сильным классическим baseline на тех же данных и метриках.

## Артефакты

Конфигурация сети, число параметров, learning curves, таблица MLP/baseline, итоговая оценка на test и анализ переобучения.

## Приёмка

Функция потерь согласована с выходом и типом задачи; test не используется для выбора эпохи; сравнение с baseline выполнено на одинаковом разбиении.
