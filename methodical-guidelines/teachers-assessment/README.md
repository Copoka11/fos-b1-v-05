# Методические указания преподавателям по оцениванию

## До начала курса

1. Проверить соответствие содержания [рабочей программе](../../docs/rpd.md).
2. Опубликовать КИМ, рубрики и правила использования генеративного ИИ.
3. Провести входную диагностику без начисления баллов.
4. Выбрать допустимые наборы данных и ограничения вычислительных ресурсов.

## В ходе курса

- оценивать наблюдаемые свидетельства из [модели измерения](../../README.md#2-модель-измерения);
- применять [единую лабораторную рубрику](rubric-laboratory.md);
- не начислять баллы дважды за один артефакт;
- проверять отсутствие утечки и воспроизводимость до обсуждения качества;
- калибровать рубрики на общих примерах;
- фиксировать причины расхождения оценок экспертов.

## После курса

Провести анализ распределения баллов, типичных ошибок и межэкспертных расхождений. По итогам реального пилота уточнить задания, дескрипторы и пороги.
