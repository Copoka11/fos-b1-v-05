# Методические указания обучающимся

1. Начните с постановки задачи, карточки данных и baseline.
2. Создайте неизменяемое test-разбиение; решения принимайте по train/CV или validation.
3. Оформляйте каждую лабораторную как воспроизводимый мини-эксперимент: вопрос → метод → результат → анализ ошибок → вывод.
4. Для проекта заранее согласуйте основную метрику и цену ошибок.
5. Сравнивайте модели на одинаковых данных и учитывайте качество, время, объяснимость и ограничения.
6. Зафиксируйте версии библиотек, seed, источник данных и команды запуска.
7. Укажите источники, вклад участников и использование генеративного ИИ.

Перед сдачей используйте:

- [лист самопроверки проекта](../../Project/attachments/student-selfcheck.md);
- [рубрику лабораторных работ](../teachers-assessment/rubric-laboratory.md);
- [чек-лист перед защитой](../../Exam/attachments/checklist-before-defense.md).

Критический дефект — утечка test, невоспроизводимый результат или неатрибутированное заимствование — требует исправления независимо от суммы баллов.
