# Промпты для работы с БЯМ

Промпты используются для проверки рассуждений и качества документации, но не для генерации неподтверждённых результатов. Все ответы модели требуют верификации.

## Проверка утечки данных

```text
Проанализируй приведённый ML-pipeline задачи классификации. Найди места, где информация из validation или test может попасть в обучение. Для каждого риска укажи механизм утечки, последствие и проверяемое исправление. Не предполагай отсутствующий код.
```

## Рецензирование дизайна эксперимента 

```text
Проверь план сравнения моделей классификации. Оцени сопоставимость разбиений, корректность метрик, настройку гиперпараметров, использование test и статистическую неопределённость. Сформулируй замечания как проверяемые гипотезы.
```

## Анализ ошибок

```text
На основе confusion matrix и таблицы ошибочных примеров предложи группы ошибок. Отдели наблюдения от гипотез. Для каждой гипотезы предложи эксперимент, который может её подтвердить или опровергнуть.
```

## Правила раскрытия использования ИИ

Студент сохраняет существенные промпты, отмечает сгенерированные фрагменты и отвечает за корректность кода, ссылок и выводов. Запрещено передавать модели персональные, конфиденциальные и лицензионно ограниченные данные.
