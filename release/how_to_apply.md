# Как применять
1. До старта выдать brief, критерии и self-check. 2. Согласовать задачу и метрику. 3. Принимать лабораторные по рубрикам. 4. Назначить по две рецензии. 5. Провести защиту. 6. Суммировать веса. 7. При критическом дефекте вернуть проект на исправление независимо от суммы. 8. После потока обновить validation/report.md.
