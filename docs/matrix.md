# Матрица КРМ

Каждый индикатор подтверждается минимум двумя средствами. Уровень целевого освоения: базовый; ML-2.3, ML-3.1—3.3 и DL-1.1—1.2 допускают демонстрацию среднего уровня.

| Роли КРМ | Индикатор | Результат обучения | Средства |
|---|---|---|---|
| Data Analyst, ML Engineer | ML-2.1 | Различает типы задач МО, выбирает и обосновывает метод решения задачи классификации. | ЛР1, текущий и рубежный контроль, Проект, Защита |
| Data Analyst, ML Engineer | ML-2.2 | Применяет предварительную обработку данных и работу с признаками. | ЛР1, текущий и рубежный контроль, Проект |
| Data Analyst, ML Engineer, ML Researcher | ML-2.3 | Оценивает качество моделей, применяет кросс-валидацию и методы работы с дисбалансом. | ЛР2, текущий и рубежный контроль, Проект, Peer-review |
| Data Analyst, ML Engineer, ML Researcher | ML-3.1 | Обосновывает применение классических методов МО с учителем и их параметров. | ЛР3, текущий и рубежный контроль, Проект, Защита |
| Data Analyst, ML Engineer, ML Researcher | ML-3.2 | Применяет классические и ансамблевые методы для достижения требуемых характеристик. | ЛР4, текущий и рубежный контроль, Проект |
| Data Analyst, ML Engineer, ML Researcher | ML-3.3 | Сопоставляет модели с аналогами и оценивает ограничения, объяснимость и ресурсоёмкость. | ЛР3, ЛР4, текущий и рубежный контроль, Проект, Peer-review, Защита |
| ML Engineer, ML Researcher | DL-1.1 | Объясняет и применяет основы обучения нейронных сетей: функции потерь, градиенты и оптимизацию. | ЛР5, текущий и рубежный контроль, Проект, Защита |
| ML Engineer, ML Researcher | DL-1.2 | Реализует MLP и выбирает архитектуру, активации и функцию потерь для классификации. | ЛР5, текущий и рубежный контроль, Проект |
| ML Engineer, ML Researcher | DL-1.3 | Применяет современные глубокие архитектуры и понимает особенности их обучения. | ЛР6, текущий и рубежный контроль, Проект |
| ML Engineer, ML Researcher | DL-1.4 | Применяет и настраивает CNN для классификации изображений. | ЛР6, текущий и рубежный контроль, Проект |
