# Критерии проекта
## Ось A. Программное решение — 60%

| Критерий | Доля внутри проекта |
|---|---:|
| Постановка задачи и baseline | 10% |
| Данные и отсутствие утечки | 15% |
| Корректность реализации и воспроизводимость | 15% |
| Экспериментальный дизайн | 10% |
| Качество и анализ ошибок | 10% |

## Ось B. Документация — 40%

| Критерий | Доля внутри проекта |
|---|---:|
| Структура и инструкция запуска | 10% |
| Обоснование методов и метрик | 10% |
| Представление результатов | 10% |
| Ограничения, этика и выводы | 10% |

Проценты в таблицах задают распределение **внутри проекта**, вес которого в итоговой оценке дисциплины составляет 25 баллов из 100.

## Дескрипторы 10-балльной шкалы
| Уровень | Дескриптор |
|---:|---|
| 10 | Полное, воспроизводимое и методически обоснованное решение; выводы подтверждены экспериментами. |
| 8 | Требования выполнены; есть небольшие недочеты без влияния на основные выводы. |
| 6 | Работоспособный минимум; анализ и обоснование частично поверхностны. |
| 4 | Фрагментарное решение; существенные пробелы, но автор понимает основные действия. |
| 0–3 | Результат не подтверждён, невоспроизводим или содержит критические методические ошибки. |
