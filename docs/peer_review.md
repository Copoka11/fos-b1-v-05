# Peer-review
Каждый проект получает две независимые рецензии. Рецензент запускает решение по инструкции и оценивает: постановку, данные, утечки, baseline, корректность валидации, метрики, сравнение моделей, анализ ошибок, воспроизводимость, документацию. Обязательны сильные стороны, риски и три конкретные рекомендации. Качество рецензии оценивается автором и преподавателем по точности, проверяемости и полезности.
