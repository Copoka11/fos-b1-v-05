# Проектная работа

Проектная работа — интегральный КИМ, проверяющий выбранные индикаторы ML-2, ML-3 и DL-1 преимущественно на уровне С.

## Постановка

Команда разрабатывает и защищает воспроизводимое решение задачи классификации табличных данных, текста или изображений. Решение должно включать:

1. постановку задачи и обоснование основной метрики;
2. анализ данных и разбиение без утечки;
3. dummy или простой содержательный baseline;
4. минимум три сопоставимые модели;
5. корректный протокол настройки и однократную итоговую оценку на test;
6. анализ ошибок, ограничений, вычислительных характеристик и рисков;
7. воспроизводимый репозиторий, отчёт и презентацию;
8. описание вклада участников и использования генеративного ИИ.

## Формат сдачи

- репозиторий с инструкцией запуска и зафиксированным окружением;
- отчёт с таблицами, графиками и выводами;
- презентация;
- две независимые рецензии и ответ авторов на замечания.

## Оценивание

- проект — **25 баллов** по [рубрике проекта](rubric-project.md);
- peer-review — **5 баллов** по [регламенту](peer-review.md);
- защита — отдельно, **10 баллов**, по материалам раздела [Exam](../Exam/README.md).

Критические дефекты: утечка test или целевой переменной, невоспроизводимый результат и неатрибутированное заимствование.
