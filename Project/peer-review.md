# Peer-review проекта

Каждый проект получает две независимые рецензии. Рецензент запускает решение по инструкции и проверяет постановку, данные, риск утечки, baseline, валидацию, метрики, сравнение моделей, анализ ошибок, воспроизводимость и документацию.

## Обязательные элементы

- проверяемые сильные стороны;
- выявленные риски и слабые места;
- три конкретные рекомендации;
- указание, какие действия и результаты были проверены;
- ролевой блок КРМ: Data Analyst, ML Engineer, ML Researcher.

## Оценивание рецензии — 5 баллов

| Критерий | Баллы |
|---|---:|
| Фактическая точность и проверяемость | 0–2 |
| Полнота проверки ключевых рисков | 0–1 |
| Конкретность и полезность рекомендаций | 0–2 |
